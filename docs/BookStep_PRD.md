# **PRD: BookStep \- 청소년 문학 도서 대출 서비스**

## **1\. 제품 개요**

### **1.1 제품명**

BookStep

### **1.2 제품 유형**

청소년 문학 도서 추천·검색·북마크·대출 예약 반응형 웹서비스 MVP

### **1.3 개발 방식**

HTML, CSS, JavaScript, LocalStorage 기반 프론트엔드 MVP

### **1.4 우선 지원 환경**

모바일 우선 반응형 웹

---

## **2\. 서비스 목적**

BookStep은 문학과 도서에 관심 있는 청소년이 자신에게 맞는 문학 도서를 쉽게 추천받고, 원하는 도서를 검색·확인한 뒤, 북마크에 담아 대출 또는 예약 신청까지 간편하게 경험할 수 있도록 지원하는 웹서비스이다.

---

## **3\. 핵심 사용자**

### **3.1 주요 사용자**

문학과 도서에 관심 있는 중·고등학생

### **3.2 사용자 특성**

* 어떤 책을 읽어야 할지 고민하는 청소년  
* 검색 결과가 많아 원하는 도서를 빠르게 찾기 어려운 사용자  
* 관심 도서를 따로 모아 관리하고 싶은 사용자  
* 모바일 환경에서 간편하게 도서 정보를 확인하고 싶은 사용자

---

## **4\. 사용자 문제**

### **4.1 핵심 문제**

사용자는 자신에게 맞는 책을 고르기 어렵고, 검색 결과가 많아 원하는 도서를 빠르게 찾기 어렵다.

### **4.2 세부 문제**

* 추천 도서 탐색의 어려움  
* 관심 장르 기반 도서 선택의 불편함  
* 검색 결과 과다로 인한 탐색 피로  
* 마음에 드는 도서를 따로 저장하기 어려움  
* 대출 가능 여부 확인과 신청 과정의 번거로움

---

## **5\. 사용자 니즈**

* 관심 장르에 맞는 도서 추천  
* 원하는 도서의 빠른 검색  
* 검색 결과 필터링 및 정렬  
* 도서 줄거리와 대출 가능 여부 확인  
* 관심 도서 북마크 저장  
* 대출·예약 신청의 간편 처리  
* 모바일 중심의 직관적 이용 경험

---

## **6\. 서비스 목표**

사용자가 회원가입 시 선택한 관심 장르를 기반으로 맞춤 추천 도서를 확인하고, 도서를 검색·필터링한 뒤, 상세 정보를 확인하여 북마크에 담거나 대출·예약 신청을 완료할 수 있는 모바일 우선 MVP를 구현한다.

---

## **7\. 사용자 흐름**

회원가입 → 관심 장르 선택 → 로그인 → 메인 화면 접속 → 추천 도서 확인 → 인기 도서 확인 → 도서 검색 → 필터·정렬 적용 → 도서 상세 확인 → 줄거리 및 대출 가능 여부 확인 → 북마크 담기 또는 대출·예약 신청 → 완료 메시지 확인

---

## **9\. 주요 기능 요구사항**

### **9.1 회원가입**

#### **기능 설명**

사용자가 이메일, 비밀번호, 닉네임, 관심 장르를 입력 또는 선택하여 간이 계정을 생성한다.

#### **입력 항목**

* 이메일  
* 비밀번호  
* 닉네임  
* 관심 장르

#### **처리 방식**

* 입력값을 LocalStorage에 저장  
* 실제 인증 절차는 구현하지 않음  
* 관심 장르는 맞춤 추천 기준으로 활용

#### **완료 조건**

* 필수 입력값 입력 시 회원가입 완료  
* 회원가입 완료 후 로그인 화면 또는 메인 화면으로 이동  
* 회원가입 완료 메시지 표시

---

### **9.2 로그인**

#### **기능 설명**

사용자가 가입 시 입력한 이메일과 비밀번호로 간이 로그인한다.

#### **입력 항목**

* 이메일  
* 비밀번호

#### **처리 방식**

* LocalStorage에 저장된 사용자 정보와 입력값 비교  
* 일치 시 로그인 상태 저장  
* 불일치 시 오류 메시지 표시

#### **완료 조건**

* 로그인 성공 시 메인 화면 이동  
* 로그인 실패 시 안내 메시지 표시

---

### **9.3 관심 장르 기반 맞춤 추천**

#### **기능 설명**

회원가입 시 선택한 관심 장르를 기준으로 메인 화면에 추천 도서를 표시한다.

#### **표시 항목**

* 도서 표지  
* 도서명  
* 저자  
* 장르  
* 간단 소개  
* 대출 가능 여부

#### **처리 방식**

* 사전 정의된 도서 데이터에서 관심 장르와 일치하는 도서 필터링  
* 메인 화면 추천 영역에 출력

#### **완료 조건**

* 로그인한 사용자의 관심 장르에 맞는 도서 목록 표시

---

### **9.4 인기 도서 표시**

#### **기능 설명**

메인 화면에서 청소년 문학 인기 도서를 보여준다.

#### **표시 항목**

* 도서 표지  
* 도서명  
* 저자  
* 장르  
* 대출 가능 여부

#### **처리 방식**

* 사전 정의된 도서 데이터 중 인기 도서 속성을 가진 항목 표시

#### **완료 조건**

* 메인 화면에서 인기 도서 목록 확인 가능

---

### **9.5 도서 검색**

#### **기능 설명**

사용자가 키워드를 입력하여 도서를 검색한다.

#### **검색 기준**

* 도서명  
* 저자  
* 장르  
* 키워드

#### **처리 방식**

* 입력 키워드와 도서 데이터 비교  
* 일치하거나 포함되는 도서 목록 표시

#### **완료 조건**

* 검색어 입력 후 관련 도서 목록 표시  
* 검색 결과가 없을 경우 안내 메시지 표시

---

### **9.6 검색 필터**

#### **기능 설명**

사용자가 검색 결과를 장르 또는 대출 가능 여부 기준으로 필터링한다.

#### **필터 기준**

* 장르  
* 대출 가능  
* 예약 가능

#### **처리 방식**

* 선택한 필터 조건에 맞는 도서만 목록에 표시

#### **완료 조건**

* 필터 선택 시 검색 결과 즉시 갱신

---

### **9.7 검색 정렬**

#### **기능 설명**

사용자가 검색 결과를 원하는 기준으로 정렬한다.

#### **정렬 기준**

* 인기순  
* 최신순  
* 제목순

#### **처리 방식**

* 선택한 정렬 기준에 따라 도서 목록 순서 변경

#### **완료 조건**

* 정렬 기준 선택 시 목록 순서 변경

---

### **9.8 도서 상세 조회**

#### **기능 설명**

사용자가 도서를 선택하면 상세 정보를 확인할 수 있다.

#### **표시 항목**

* 도서 표지  
* 도서명  
* 저자  
* 장르  
* 줄거리  
* 대출 가능 여부  
* 대출 버튼  
* 예약 버튼  
* 북마크 버튼

#### **처리 방식**

* 선택한 도서 ID를 기준으로 상세 정보 표시

#### **완료 조건**

* 도서 목록에서 도서 선택 시 상세 화면 표시

---

### **9.9 대출 신청**

#### **기능 설명**

대출 가능한 도서에 대해 사용자가 대출 신청을 할 수 있다.

#### **처리 방식**

* 대출 가능 도서일 경우 대출 신청 버튼 활성화  
* 버튼 클릭 시 대출 신청 완료 메시지 표시  
* 신청 정보는 LocalStorage에 저장 가능

#### **완료 조건**

* 대출 신청 완료 메시지 표시

---

### **9.10 예약 신청**

#### **기능 설명**

대출 중이거나 예약 가능한 도서에 대해 사용자가 예약 신청을 할 수 있다.

#### **처리 방식**

* 예약 가능 도서일 경우 예약 버튼 활성화  
* 버튼 클릭 시 예약 완료 메시지 표시  
* 예약 정보는 LocalStorage에 저장 가능

#### **완료 조건**

* 예약 신청 완료 메시지 표시

---

### **9.11 북마크**

#### **기능 설명**

사용자가 관심 도서를 북마크에 저장할 수 있다.

#### **처리 방식**

* 북마크 버튼 클릭 시 해당 도서 정보를 LocalStorage에 저장  
* 이미 저장된 도서는 북마크 해제 가능  
* 북마크 목록에서 저장된 도서 확인 가능

#### **완료 조건**

* 북마크 추가 및 해제 가능  
* 새로고침 후에도 북마크 유지

---

## **13\. UI 요구사항**

* 모바일 우선 레이아웃 적용  
* 청소년 사용자가 이해하기 쉬운 직관적 화면 구성  
* 도서 카드 중심의 목록 UI 적용  
* 주요 버튼은 명확하게 구분  
* 검색창은 화면 상단에 배치  
* 도서 상세 화면에서 대출·예약·북마크 버튼을 쉽게 찾을 수 있도록 배치  
* 완료 메시지는 즉시 확인 가능하도록 표시  
* PC 화면에서도 깨지지 않는 반응형 구조 적용

## **17\. 테스트 시나리오**

### **17.1 회원가입 테스트**

* 이메일, 비밀번호, 닉네임, 관심 장르 입력  
* 회원가입 버튼 클릭  
* 회원가입 완료 메시지 확인  
* LocalStorage 저장 여부 확인

### **17.2 로그인 테스트**

* 가입한 이메일과 비밀번호 입력  
* 로그인 버튼 클릭  
* 메인 화면 이동 확인

### **17.3 추천 도서 테스트**

* 관심 장르와 일치하는 도서가 메인에 표시되는지 확인

### **17.4 검색 테스트**

* 도서명 키워드 입력  
* 관련 도서 목록 표시 확인

### **17.5 필터 테스트**

* 장르 필터 선택  
* 해당 장르 도서만 표시되는지 확인

### **17.6 정렬 테스트**

* 인기순, 최신순, 제목순 선택  
* 목록 순서 변경 확인

### **17.7 상세 조회 테스트**

* 도서 카드 클릭  
* 상세 정보 표시 확인

### **17.8 북마크 테스트**

* 북마크 버튼 클릭  
* 북마크 목록 저장 확인  
* 새로고침 후 유지 확인

### **17.9 대출 신청 테스트**

* 대출 가능 도서 선택  
* 대출 신청 버튼 클릭  
* 완료 메시지 확인

### **17.10 예약 신청 테스트**

* 예약 가능 도서 선택  
* 예약 신청 버튼 클릭  
* 완료 메시지 확인

## **18\. 완료 정의**

BookStep MVP는 사용자가 LocalStorage 기반으로 간이 회원가입과 로그인을 수행하고, 관심 장르 기반 추천 도서와 인기 도서를 확인하며, 도서를 검색·필터·정렬한 뒤 상세 정보를 조회하고, 북마크 저장 및 대출·예약 신청 완료까지 경험할 수 있을 때 완료된 것으로 본다.
